# agent-society-sim

> 基于大语言模型的多智能体**社会模拟**引擎。每个智能体拥有对外的公开人设、与仅自身可见的私密目标和秘密,被关进同一段时间、同一组场景里自主交互;开发者以"上帝视角"观察由此**涌现**的群体行为与关系演变。

引擎与世界设定解耦:**引擎是通用的,世界设定是可替换的数据**(`content/scenarios/*.yaml`)。

---

## 核心特性

| 维度 | 能力 |
|---|---|
| **表里双层人格** | 每个 agent 分离存储对外表现(公开人设)与内心世界(私密目标 / 秘密),如常人般表里有别 |
| **第一人称内独白 prompt** | 按"稳定 → 易变"七层拼装——身份事实 · 信念透镜 · 意图 · 记忆 · 处境 · 此刻骤变 · 冲动,借近因效应让最新刺激获得最高注意力权重 |
| **工具化决策** | 6 种动作(`speak` / `act` / `move` / `silence` 外显 + `think` / `plan` 内心)以 OpenAI 兼容的 tool schema 在 API 层强制,LLM 只能输出合法行为 |
| **情景—语义记忆** | 以第一人称情景记忆为唯一源,分两支:滚动压缩兜住物理长度、周期反思蒸馏出语义记忆(自身 belief + 对他人的量化情感关系),语义回流驱动后续决策 |
| **观察者注入** | 外部观察者可向世界注入三类要素——客观事实、转瞬冲动、持久信念(belief),分别落进不同层、存活不同时长 |
| **叙事时钟** | 把单调递增的 tick 翻译成"第几天 · 哪个时段"的分层日历,渲染时段氛围;终局事件给全场施加倒计时压迫 |
| **场景即数据** | 角色 / 地点 / 日历皆为可替换的 YAML 数据——换一份文件即换一个世界设定|
| **事件溯源式持久化** | 每拍自动落盘:append-only 事件流 + 元状态 + 单 agent 快照,支持跨进程重启续演 |

## 架构

系统按**同心圆**组织:越往内越稳定通用,依赖只能由外向内——外层认识内层,内层对外层一无所知。

### 静态视图

```
 ╭─────────────────────────────────────────────────────────────────╮
 │  交互层 · Interaction              〔最外:驱动输入 · 呈现输出〕   │
 │  Console 解析输入   CLISink 渲染单行   (TUI / Web / 回放 可平替)   │
 │  ╭══════════════════ 端口膜 · Ports ══════════════════════════╮  │
 │  ║  Conductor(控制力向内 ↓)        Signal / SimSink(事件向外 ↑)║  │
 │  ║  ╭───────────────────────────────────────────────────────╮ ║  │
 │  ║  │  World · 世界的运转            〔引擎:编排 · 推进世界〕  │ ║  │
 │  ║  │  turn_engine 回合编排·状态提交  perception 可见域 · director│ ║  │
 │  ║  │  ╭─────────────────────────────────────────────────╮  │ ║  │
 │  ║  │  │  Agent · 角色的心智         〔感知→决策→沉淀〕    │  │ ║  │
 │  ║  │  │  brain 决策 · prompts · tools · memory 三级记忆   │  │ ║  │
 │  ║  │  │  ╭───────────────────────────────────────────╮  │  │ ║  │
 │  ║  │  │  │  Core · 唯一事实源           〔纯数据〕     │  │  │ ║  │
 │  ║  │  │  │  WorldState · Agent(公开/私密) · Event ·    │  │  │ ║  │
 │  ║  │  │  │  Location · Calendar · 跨层契约;不依赖任何人│  │  │ ║  │
 │  ║  │  │  ╰───────────────────────────────────────────╯  │  │ ║  │
 │  ║  │  ╰─────────────────────────────────────────────────╯  │ ║  │
 │  ║  ╰───────────────────────────────────────────────────────╯ ║  │
 │  ╚═════════════════════════════════════════════════════════════╝  │
 ╰─────────────────────────────────────────────────────────────────╯

   依赖规则:箭头只能由外向内 →·  Core 处最内、最稳定,对外层一无所知
   运行时  :控制力由外向内驱动 ↓ ,  事件由内向外冒泡 ↑(穿过端口膜)
   ┄ 横切适配器(不占某一环,被各环按需接入):LLM 接入 · 持久化 · 配置 · 场景装配
```

**各环职责:**

- **Core**——引擎只认的通用数据与跨层契约(`WorldState` / `Agent` / `Event` / `Calendar` / `Perception`),不依赖任何上层。
- **Agent**——单个角色"感知 → 决策 → 沉淀"的认知闭环:`brain` 决策 + `memory` 记忆,只依赖 Core。
- **World**——对世界的操控:`turn_engine` 推进回合、`perception` 框定 agent 可见范围、`director` 注入外部事件。

### 动态视图 · 一个 tick 的执行流程

一个 tick 内引擎如何驱动各 agent。系统本质是**事件溯源**:`event_log` 是 append-only 的唯一历史,世界可变状态是其 fold 结果。

```
  Conductor.step()                                      〔控制面 · 向内驱动〕
       │
       ▼
  turn_engine.run_turn
       │
       ├─① director.sparks_for() ──► 火种 Event ──┐
       │                                          │ append
       │                                          ▼
       │                              ╔═══════════════════════════╗
       └─② 逐个 agent(顺序推进):    ║   event_log(append-only)  ║
                                      ║   ── 世界的唯一历史 ──     ║
           perceive ◄── 按可见范围裁剪 ──╢  WorldState 的可变态 = fold ║
               │  读                   ╚═══════════╤═══════════════╝
               ▼                                   ▲
           brain.decide                            │ append(状态提交)
               │  拼第一人称 prompt                 │
               ▼          ┌───────────┐            │
           (调用)─────────► LLM 适配器 │            │
               │  ◄───────└───────────┘            │
               │  tool_calls → Event[](意图)        │
               ▼                                    │
           apply_event ── 将行为意图提交进世界状态 ──┘
               │  (MOVE 展开为 离场 / 到场 两条)
               ▼
           remember → reflect → compress     沉淀进 agent 私有记忆
               │
               ▼  每件已提交的事件
           SimSink.emit(Signal) ──► CLISink 渲染            〔数据面 · 向外冒泡〕

   ╎ 持久化:每拍把 event_log 增量 append + agent 快照覆写到 data/
```

> **关键时序**:`perceive` 的时间边界是"自己上次动作之后"而非"本 tick"——所以顺序推进时,后行动者能立即感知到前行动者本拍刚说的话;反思(`reflect`)刻意先于压缩(`compress`),好趁情景被溶解前从最丰富的那版经历里蒸馏。

## 记忆子系统

记忆只有一个源头:**第一人称情景记忆**。在它之上做两件事——一是滚动压缩、兜住物理长度(仍是情景);二是周期反思,从情景中**蒸馏出语义记忆**。语义记忆分两个侧面:对世界与自我的 `belief`(笃定看法),以及由此对他人形成的**量化情感关系**(定性印象 + -100..100 好感分)。蒸馏出的语义再**回流进下一拍的独白**,成为看人看事的透镜,闭环由此合上:

```
   情景记忆 · 第一人称经历流
   「我听到 X 说… / 我看到 Y… / 我心里掠过… / 我去了…」
        │
        ├─ 滚动压缩:超阈值时旧情景压成单条「梗概」+ 保留近端原文(兜住物理长度)
        │
        └─ 周期反思:从情景中蒸馏语义记忆 ──────────────┐
                                                       ▼
                                  ╔═════════ 语义记忆 ═════════╗
                                  ║  belief        自身笃定看法 ║
                                  ║  量化情感关系   对他人的定性 ║
                                  ║                印象 + 好感分 ║
                                  ╚═════════════╤══════════════╝
                                                │ 回流:作为透镜垫进下一拍独白
                                                ▼
                                       影响后续的感知与决策
```

## 运行与部署

### 环境要求

- Python ≥ 3.12
- 包管理:[`uv`](https://github.com/astral-sh/uv)

### 安装

```bash
git clone https://github.com/yzheeng/agent-society-sim
cd agent-society-sim
uv sync          
```

### 配置 LLM

在 `config.json` 的 `llm.active` 切换 profile:

```jsonc
{
  "llm": {
    "active": "remote",          // remote | local
    "profiles": {
      "remote": { "model": "deepseek-v4-flash", "base_url": "https://api.deepseek.com",
                  "api_key_env": "DEEPSEEK_API_KEY" },
      "local":  { "model": "qwen3-30b-a3b", "base_url": "http://localhost:1234/v1",
                  "params": { "temperature": 0.7, "presence_penalty": 1.0 } }
    }
  }
}
```

### 运行

```bash
uv run run.py
```

### 选择 / 编写场景

`content/scenarios/` 下每个 `.yaml` 是一份独立的世界设定。把 `run.py` 里的 `SCENARIO` 改成其中任一文件名即可切换:

```python
SCENARIO = "sample"   # 对应 content/scenarios/sample.yaml
```
